Limitations of an Array:-
=-=-=-=-=-=--=--=-=--=-=
1. Fixed Size:-
- Once an array is created, its size cannot be changed.
- You must know the size in advance.
- Problem: If you need more space, you must create a new array and copy elements.

2. Homogeneous Data
- Arrays can only store elements of the same data type (except Object[] arrays).

3. No Built-in Methods for Manipulation
- Arrays lack rich utility methods like adding, removing, or searching elements.

4. Memory Waste (for Large Arrays)
- If you declare a large array but use only a few elements, memory is wasted.
- Fixed size can lead to under-utilization.

=-=-==-=-=-=-=-=-=-=-=-=-=-=

public class DynamicArray {
    Object arr[];
    int size;
    int capacity;
    DynamicArray(){
        capacity =10;
        arr = new Object[capacity];
        size=0;
    }
    DynamicArray(int capacity){
        this.capacity =capacity;
        arr = new Object[capacity];
        size=0;
    }
    void increaseCapacity(){
        int newCapa = capacity+(capacity/2);
        Object[] newArr = new Object[newCapa];
        for(int i=0; i<size; i++){
            newArr[i] = arr[i];
        }
        arr = newArr;
        capacity = newCapa;
    }
    void add(Object ele){
        if(size==capacity){
            increaseCapacity();
        }
        arr[size] = ele;
        size++;
    }
    int searchEle(Object ele){
        for(int i=0; i<size; i++){
            if(arr[i]==ele){
                return i;
            }
        }
        return -1;
    }
    int getSize(){
        return size;
    }
    void display(){
        for(int i=0; i<size;i++){
            System.out.print(arr[i]+" ");
        }
        System.out.println();
    }
}
public class Demo {
    public static void main(String[] args) {
        DynamicArray da = new DynamicArray();
        for(int i=0; i<20; i++){
            da.add(i);
        }

        da.display();
        System.out.println(da.getSize());
        System.out.println(da.searchEle(10));
        System.out.println(da.searchEle(30));
    }
}

Collection :-
=-=-=-=-=-=-=
A Collection in Java is a framework that provides a set of classes and interfaces to store, manage, and manipulate groups of objects dynamically.
							or
A collection is a group of individual data that are grouped together to form a single unit. Collections are containers that hold multiple data items as a single unit.

The two root interfaces of java collection class are:
	- Collection interface(java.util.collection)
	- Map interface(java.util.Map)


Collection Framework in Java:-
=-=-=-=-=-=-=-=-=-=--==-=-=-=
The Java Collection Framework (JCF) is a set of classes and interfaces that provide predefined data structures to store, retrieve, and manipulate groups of objects efficiently. It is part of the java.util package.


Hierarchy of Collection F/W:-

Basic Functionalities of List, Set, and Queue Interface:-
=-=-=-=-=-=-=-==-=-=-=-=-=-=-==-=--=-=-=-=--=-=-=--=-=-=-=

1. List Interface
------------------
Functionality
- Ordered collection (maintain insertion order)
- Duplicates allowed
- Index-based access (0,1,2…)
- Null values allowed

Use Case
- Order important ho
- When need duplicate values
- Element need access by Index


2. Set Interface
---------------
Functionality
- No duplicates allowed
- Order generally not guaranteed (except some implementations)
- No index-based access
- Null allowed (usually only one)

Use Case
- Need unique elements 
- Avoid duplicate 

3. Queue Interface
------------------
Functionality
- FIFO (First In First Out)
- Elements process in Order
- Mainly used for processing tasks
- No index access

Use Case
- Use in Line/queue system (like ticket counter)
- Need Task scheduling

1. ArrayList:-
=-=-=-=-=-=-=
- ArrayList is a resizable array implementation of the List interface.
- It allows dynamic growth, unlike arrays which have fixed size.
- It maintains insertion order, allows duplicate elements, and nulls.
- Implements:
	- List<E>
	- Cloneable
	- Serializable
- RandomAccess (supports fast index-based access)

Internal Storage of ArrayList
-----------------------
- Internally, ArrayList uses a dynamic array, specifically an Object[] array called elementData.
- This array is not exposed directly; all operations like add, get, remove work through methods.

Default Size and Capacity
--------------------
Default initial capacity (Java 8+): 10

- ArrayList<Integer> list = new ArrayList<>(); // capacity = 10

If you provide an initial capacity:

- ArrayList<Integer> list = new ArrayList<>(20); // capacity = 20

Key distinction:
----------------
size() → number of elements currently in list
capacity → length of the internal array

How ArrayList Grows (Resizing Behavior)
---------------------------------
- When an element is added and the internal array is full, ArrayList resizes:
	- New Capacity Formula :
		newCapacity = oldCapacity + (oldCapacity/2);

- This means capacity increases by 50% each time.
- Steps:
	- Check if size < capacity → add element directly
	- If full → create a new array with 1.5× capacity
	- Copy old elements → new array
	- Add the new element
Example:
- Default capacity = 10
- Add 11th element → new capacity = 15
- Add 16th element → new capacity = 22

**Note: In Java 6-7, capacity increased by 100%, i.e., doubled.

2. LinkedList:-
=-=-=-=-=-=-=
- LinkedList is a doubly linked list implementation of the List interface.
- It allows dynamic memory allocation (no fixed size limitation).
- It maintains insertion order, allows duplicate elements, and nulls.
- Implements:
	- List<E>
	- Deque<E>
	- Cloneable
	- Serializable

- Does NOT implement RandomAccess (slow index-based access)

Internal Storage of LinkedList
------------------------------
- Internally, LinkedList uses a doubly linked list.
- Each element is stored in a Node.
- Each node contains:
	- data (element)
	- reference to next node
	- reference to previous node

Default Size and Capacity
-------------------------
- LinkedList does NOT have capacity concept like ArrayList.
- It grows dynamically as elements are added.

Key distinction:
----------------
- size() → number of elements currently in list
- capacity →  Not applicable in LinkedList

How LinkedList Grows (Memory Behavior)
--------------------------------------
- No resizing like ArrayList.
- Each element is stored in a new node.

Steps:
- Create a new node
- Link it with previous and next node
- Update pointers

Example:

Add element → new node created
No copying of old elements required

Creation of a LinkedList:-
=-=-=-=-=-=-=-=-=-=-=-=-=

1. Using default constructor without any type safety:-
2. Using default constructor with type safety:-
Example:-
	LinkedList li = new LinkedList();
	LinkedList<String> li1 = new LinkedList<>();

Methods Specific to LinkedList Class Only:-
=-=-=-=-=-=-=-=-=-=-==-=-=-=-=-=-=-=-=-=
Specific for Linked List:-
-------------------------
- void addFirst(E e) → Inserts element at the beginning.
- void addLast(E e) → Inserts element at the end.
- E getFirst() → Returns first element.
- E getLast() → Returns last element.
- E removeFirst() → Removes first element.
- E removeLast() → Removes last element.
- E push(E e) → Adds element at front.(Can help to implement Stack)
- E pop() → Removes element from front.(Can help to implement Stack)

Methods from Deque:-
-------------------
- boolean offer(E e) → Adds element at end (queue style).(Can help to implement Queue)
- boolean offerFirst(E e) → Adds element at beginning.(Can help to implement deque)
- boolean offerLast(E e) → Adds element at end.(Can help to implement deque)
- E peek() → Retrieves head without removing.
- E poll() → Retrieves and removes head. (Can help to implement Queue)

Example code:-
=-=-=-=-=-=-=
public class LinkedListExample {
    public static void main(String[] args) {
        LinkedList<String> li1 = new LinkedList<>();
        
        //Adding elements at the last Methods
        li1.add("Krish");
        li1.offer("Himanshu");
        li1.addLast("Satyam");
        li1.offerLast("Koushik");
        System.out.println(li1);

        //Adding elements at the first Methods
        li1.addFirst("Nikita");
        li1.push("Khushi");
        li1.offerFirst("Sargam");

        System.out.println(li1);

        //Other Methods

        System.out.println(li1.getFirst());
        System.out.println(li1.getLast());
        System.out.println(li1.peek());
        System.out.println(li1.removeFirst());
        System.out.println(li1.removeLast());
        System.out.println(li1.poll());

        System.out.println(li1);
    }
}

Why we can use LinkedList as Queue, Deque, and Stack Operations:-
--------------------------------------------------------
- LinkedList is a data structure that allows fast insertion/deletion at both ends, which perfectly matches the Deque contract.
- ArrayList cannot efficiently implement Deque because adding/removing from head requires shifting all elements.

Basic difference btw ArrayList and LinkedList:-
------------------------------------------------
Feature			ArrayList								LinkedList
-------                 ---------                       ----------
Data Structure		Dynamic Array						Doubly Linked List
Memory Storage		Continuous (array)					Non-contiguous (nodes)
Access Time			Fast (O(1))							Slow (O(n))

Why we need LinkedList when we have ArrayList:-
=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=
Even though ArrayList is faster in many cases, LinkedList is useful in specific scenarios :-
------------------------------------------------------------------------------------------
- Frequent Insertions/Deletions
- When Data Size Changes Frequently
- No resizing overhead (ArrayList needs copying)
- When Insertion in Middle is Frequent, Faster than ArrayList (no shifting of all elements)



Vector
=-=-=-=
- Vector is a resizable array implementation of the List interface.
- It is synchronized (thread-safe) unlike ArrayList.
- Maintains insertion order, allows duplicate elements, and nulls.
- Implements:
	- List<E>
	- Cloneable
	- Serializable

Internal Storage of Vector
---------------------------
- Internally, Vector uses an Object[] array to store elements.
- Initial capacity can grow dynamically.
- Default Size and Capacity
	- Default initial capacity = 10
		- Vector<Integer> v = new Vector<>(); // capacity = 10
	- If you provide an initial capacity:
		- Vector<Integer> v = new Vector<>(20); // capacity = 20

Key distinction:
----------------
- size() → number of elements currently in vector
- capacity → length of the internal array

How Vector Grows (Resizing Behavior)
-------------------------------------
- When elements exceed capacity, Vector doubles the capacity (100% increase).
Steps:
- Create a new array with double size
- Copy old elements → new array
- Add new element

Example:

- Default capacity = 10
- Add 11th element → new capacity = 20
- Add 21st element → new capacity = 40

Creation of a Vector
=-=-=-=-=-=-=-=-=-=

- Using default constructor:
	Vector<Integer> v = new Vector<>();

- Using constructor with initial capacity:
	Vector<Integer> v = new Vector<>(50);

- Using constructor with another collection:
	Vector<Integer> v = new Vector<>(Arrays.asList(1,2,3));

Methods Specific to Vector Class Only
=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=

- void ensureCapacity(int minCapacity) → Increases internal array capacity if needed
- void trimToSize() → Trims capacity to match current size
- synchronized methods for thread-safety, e.g., add(), remove(), get(), etc.


Stack
=-=-=
- Stack is a LIFO (Last In First Out) data structure implemented as a subclass of Vector.
- Maintains insertion order, allows duplicates and nulls.
- Implements:
	- List<E>
	- Cloneable
	- Serializable

Internal Storage of Stack
--------------------------
- Internally, Stack uses Vector for storage.
- Elements are stored in an array (dynamic).

Default Size and Capacity
-------------------------
- Inherits Vector’s default capacity = 10
- Resizes automatically when full (doubles capacity).

How Stack Works (Memory Behavior)
----------------------------------
- Stack follows LIFO (Last In First Out).
- Push → adds element at top
- Pop → removes element from top
- Peek → returns element at top without removing

Creation of a Stack
=-=-=-=-=-=-=-=-=-=
- Using default constructor:
	- Stack<Integer> s = new Stack<>();

Methods Specific to Stack Class Only
=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=
- void push(E e) - Adds element at top of stack
- E pop() - Removes and returns element from top
- E peek() - Returns element at top without removing
- boolean empty() - Returns true if stack is empty
- int search(Object o) - Returns 1-based position from top; -1 if not found


Example Code:-
------------

Iterator interface:
===================
--This interface gives us a standardize way of accessing the elements from any Collection object.
--while iterating objects by iterator we can perform remove operation in addition to read operation.
--It is a Universal cursor (Works with List, Set and Queue).

Methods in Iterator interface:
-----------------------------

public boolean hasNext();  // check if next element exists
public E next();           // get next element
void remove();      // remove current element safely




-To get the Iterator object, we need to call iterator() method on any Collection object.

Example:
	Iterator itr= al.iterator();

--It creates an iterator object that points to the same ArrayList object.


Example:
-------
public class Main {
	public static void main(String[] args) {
		List<String> list = new ArrayList<>(List.of("a","b","c","d","e","f"));
		//find the element "b" AND "c" from list and remove it.
		Iterator itr = list.iterator();
		while(itr.hasNext()) {
			System.out.print(itr.next()+" ");
		}
	}
}
Output:-
-------
a b c d e f 


Note: If we are trying to remove the element while iterating for enhanced for loop, It will throw the ConcurrentModificationException.

limitations of Iterator:
----------------------

1. By using Iterator we can always move forward direction, but can't move backward direction, it is a single direction cursor.

2. By using the Iterator we can perform only read and remove operation, but we can't perform the replace and add operation.

Note: ListIterator is the most powerful cursor but its limitation is, It is not a universal cursor, we can use it only with List objects not with the Set or Queue objects.

1. HashSet:-
=-=-=-=-=-=
- HashSet is a class that implements the Set interface.
- It does not allow duplicate elements.
- It does not maintain insertion order.
- It allows one null value.
- Internally uses a HashMap.
- Implements:
	- Set<E>
	- Cloneable
	- Serializable

Internal Working of HashSet
---------------------------
- It is internally implemented using a HashMap.
- In HashSet, elements are stored as keys of HashMap and a dummy value is used. Like key-value pair but at the place value used a dummy value.
- When an element is added, first its hashCode() method is called to generate their hashCode value.
- The hashCode() method generates an integer value that helps in finding the bucket location.
- Based on this hashCode, the element is placed into a specific bucket.
- If the bucket is empty, the element is stored directly.
- If the bucket already contains elements, a collision occurs.
- In case of collision, equals() method is used to compare the new element with existing elements in that bucket.
- If equals returns true, the element is considered duplicate and is not added.
- If equals returns false, the element is added in the same bucket.

- So, hashCode() is always checked first to find the location.
- equals() is checked only when multiple elements have the same hashCode.

- If both hashCode is same and equals returns true, the element is treated as duplicate.
- If hashCode is same but equals returns false, both elements are stored.

- HashSet allows only one null value because all nulls have the same hashCode and are considered equal.

Default Capacity and Load Factor
--------------------------------
- Default initial capacity: 16
- Load factor: 0.75 (Capacity expansion happens when the number of elements exceeds 75% of the current capacity. This threshold is called the load factor limit.)
- Capacity doubled after reached at Load factor.
- During resizing, all existing elements are rehashed and placed into new buckets based on the new capacity.
- This process is called rehashing and it helps maintain efficient performance of HashSet operations.

Creation of HashSet:-
=-=-=-=-=-=-=-=-=-=-=-=
- Without type safety:
	Set s = new HashSet();

- With type safety:
	Set<String> s = new HashSet<>();

- With initial capacity:
	Set<String> s = new HashSet<>(20);

2. Removing Elements
- boolean remove(Object o); → removes specific element
- boolean removeAll(Collection<?> c); → removes all matching elements
- boolean retainAll(Collection<?> c); → keeps only common elements
- void clear(); → removes everything

3. Searching / Checking
boolean contains(Object o); → checks element
boolean containsAll(Collection<?> c); → checks collection
boolean isEmpty(); → check empty or not

Example with user Define Class:-
=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=
- As we know, a HashSet is a collection that does not allow duplicate elements.
- It stores elements using a hashing mechanism, which internally relies on two important methods from the Object class:
	- hashCode()
	- equals()

- When we store user-defined objects in a HashSet, we must override these methods to ensure correct duplicate detection.
- First object will use hashCode() to find the Bucket location and after that if there is no element in Bucket it will store the data directly or if there is any element, then first it will check the content using equals() method after, if equals() method returns false then store data inside the Bucket or equals() returns true then data will not be added due to duplicate data.

Why Both Methods Must Be Overridden Together:-
=-=-===-=-==-=-=--==--=-==-=--==--=-==-=-=-=-

- It is mandatory to override both methods because:

Rule 1
-------
- If two objects are equal using equals(),
- then their hashCode() must also be equal.

Rule 2
-------
- If only equals() is overridden
- but hashCode() is not overridden:
	- HashSet may place objects in different buckets
	- Duplicate detection fails

LinkedHashSet:-
=-=-=-=-=-=-=-=-
- LinkedHashSet is a class that implements the Set interface.

- It maintains insertion order of elements.
- It allows one null value.

Internal Working of LinkedHashSet
---------------------------------
- LinkedHashSet internally uses LinkedHashMap.
- LinkedHashMap internally uses HashMap + Doubly Linked List.
- HashMap part stores elements using hashing (hashCode)(same as above we have studied in HashSet).
- At the same time, elements are linked using a doubly linked list.
- This doubly linked list maintains insertion order.
- Iteration over LinkedHashSet happens through the linked list, not the hash table.
- Duplicate elements are prevented using equals() method.
- Removal deletes element from both hash table and linked list.
- Searching is fast because hashing is used (O(1) average time).


Default Capacity of LinkedHashSet
----------------------------------

- Same as HashSet / HashMap.
	- Default Capacity = 16
	- Default Load Factor = 0.75

- Meaning:
	- Initially 16 buckets created.
	- When 75% full → resize happens.

Revisiting Collections.sort() method:-
=-=-=-==-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=

The Collections.sort() method is used to sort elements of a List either:
- According to their natural ordering, or
- According to a custom ordering defined by the programmer. 

Internal Working of Collections.sort():-
---------------------------------------

When Collections.sort(list) is called:
- Elements of the list are compared.
- Comparison is done using:
	- compareTo() (Comparable) (Natural sorting)
	- compare() (Comparator) (Custom sorting)
- Elements are rearranged into sorted order.
- An efficient sorting algorithm is used internally.

Requirement for Sorting:-
-----------------------

- To use Collections.sort():
	- Elements must implement Comparable
		OR
	- A Comparator must be provided.
- If neither Comparable nor Comparator is available:
	- A runtime exception occurs
	 (ClassCastException).

Comparable Interface:-
=--=-=-=-=-=-=-=-=-=
- The Comparable interface in Java is used to define the natural ordering of objects.
- It is part of the java.lang package and is widely used in sorting operations.
- When objects of a user-defined class need to be sorted, the class must implement the Comparable interface and override its compareTo() method.
- The Comparable interface contains only one method, making it a single-method interface.
- Method Declaration
	public interface Comparable<T> {
    		int compareTo(T obj);
	}
The Comparable interface is used to:
- Define natural sorting order of objects
- Allow objects to be sorted automatically
- Enable sorting using methods like:
	Collections.sort()
	Arrays.sort()
	TreeSet
	TreeMap

- Without Comparable, Java cannot determine how objects should be ordered.

- What is Natural Ordering?
	- Natural ordering means the default sorting order defined by the class itself.

Examples:

Class		|  Natural Order
Integer		|  Numerical order
String		|  Alphabetical order
Character	|  Alphabetical order

** Note:- Built-in classes like Integer, String, and Double already implement Comparable.

Understanding compareTo() Method:-
---------------------------------
The compareTo() method is used to compare current object with another object.

Syntax
	public int compareTo(T obj)

The method returns:
------------------
Return Value			Meaning
Negative value			Current object is smaller
Zero				Objects are equal
Positive value			Current object is greater

Meaning:
--------
If result < 0 → smaller
If result = 0 → equal
If result > 0 → greater

When sorting is performed using:
---------------------------------
	Collections.sort(list);

Java internally:
----------------
- Calls compareTo() method
- Compares objects
- Rearranges elements
- Produces sorted order
- So sorting depends completely on the compareTo() logic.

Steps to Implement Comparable Interface
-----------------------------------------
To implement Comparable in a user-defined class:

Step 1
- Implement Comparable interface
	class Student implements Comparable<Student>

Step 2
- Override compareTo() method
	@Override
	public int compareTo(Student s) {
    		return this.id - s.id;
	}

Step 3
- Use sorting method
	Collections.sort(list);

Comparator Interface
=-=-=-=-=-=-=--=-=-=
- The Comparator interface in Java is used to define a custom sorting order for objects.
- It belongs to the java.util package and is widely used when multiple sorting criteria are required.
- Unlike Comparable, which defines natural ordering inside the class, Comparator allows sorting outside the class, making it more flexible and reusable.
- Comparator is a functional interface (contains one abstract method), which means it can also be implemented using lambda expressions.

Method Declaration
-------------------
	public interface Comparator<T> {
    		int compare(T o1, T o2);
	}
Purpose of Comparator Interface
--------------------------------
- Define custom sorting logic
- Provide multiple sorting orders
- Sort objects without modifying the original class
- Compare two different objects
- Provide flexible sorting strategies

Comparator is commonly used with:
- Collections.sort()
- List.sort()
- Arrays.sort()
- TreeSet
- TreeMap

Understanding compare() Method
--------------------------------
- The compare() method compares two objects and determines their order.

Syntax:-
	public int compare(T o1, T o2)

Return Values of compare()
--------------------------
Return Value		Meaning
Negative value		First object is smaller
Zero				Objects are equal
Positive value		First object is greater

Meaning:
--------
- If result < 0 → o1 comes before o2
- If result = 0 → both equal
- If result > 0 → o1 comes after o2

***Sorting behavior depends entirely on the compare() method logic.

Steps to Implement Comparator Interface
---------------------------------------
Step 1 — Create Comparator Class
	class NameComparator implements Comparator<Student>

Step 2 — Override compare() Method
	@Override
	public int compare(Student s1, Student s2) {
    		return s1.name.compareTo(s2.name);
	}
	
Step 3 — Use Comparator in Sorting
	Collections.sort(list, new NameComparator());

Key Difference between Comparable and Comparator:-
=-=-=-=-=-=-=-=-=-=-=-==-=-=-=-=-=-=-=-=-=-=-=-=
Comparable → Natural Sorting
-----------
- Sorting logic is defined inside the class
- Used when only one default sorting order is needed
- Uses:
	compareTo()


Comparator → Custom Sorting
----------
- Sorting logic is defined outside the class
- Used when multiple sorting orders are needed
- Uses:
	compare()

If Comparator is provided → Comparator used  
If a Comparator is not provided → Comparable is used

TreeSet:-
=-=-=-=-=-=
- TreeSet is a class that implements the NavigableSet interface.
- It does not allow duplicate elements.
- It maintains sorted order (ascending by default).
- It does not allow null values.
- Internally uses a Red-Black Tree (Self-balancing Binary Search Tree).
- Elements are stored in sorted order automatically.
- Implements:
	- NavigableSet<E>
	- SortedSet<E>
	- Set<E>
	- Cloneable
	- Serializable

Internal Working of TreeSet:-
-----------------------------
- TreeSet stores elements using a self-balancing binary search tree called Red-Black Tree.
- When an element is added, TreeSet does not use hashCode().
- Instead, TreeSet uses:
	- compareTo() method (if Comparable used)
			OR
	- compare() method (if Comparator used)

- When an element is inserted, TreeSet compares the new element with existing elements.
- If the tree is empty, the element becomes the root node.
- If the tree already contains elements, comparison starts from the root.
- Based on comparison result:
	- If result < 0 → element goes to left side
	- If result > 0 → element goes to right side
	- If result = 0 → element is treated as duplicate and not added
- After insertion, TreeSet automatically balances the tree using Red-Black Tree rules.
- Because of balancing, search, insert, and delete operations are efficient.
 

*******Note***********
- TreeSet internally uses the Comparable interface for natural sorting of elements and the Comparator interface for custom sorting.
- Whenever an element is added into TreeSet, it compares elements using compareTo() (Comparable) or compare() (Comparator) methods to maintain sorted order and to prevent duplicate elements. 
- Built-in classes like Integer, String, Double already implement's Comparable Interface.
- But for user defined classes we need to implement Comparable interface for natural sorting of that particular class or provide Comparator object (Comparator implemented class object) for custom sorting.
**********************

=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=
1. Default Constructor
	TreeSet<Integer> set = new TreeSet<>();

2. Collection Constructor 
		ArrayList<Integer> list = new ArrayList<>(List.of(1,1,1,2,3,42,4,1,6,7));
		TreeSet<Integer> set = new TreeSet<>(list);

3. Comparator Constructor

		TreeSet<Integer> set = new TreeSet<>(new NameComparator());

Capacity of TreeSet:-
---------------------
- TreeSet does not have any default capacity. In TreeSet, capacity does not increase like array-based collections. Whenever a new element is added, a new node is dynamically created in the tree structure, allowing TreeSet to grow automatically as elements are inserted.

Specific Methods from NavigableSet Interface:-
---------------------------------------------
These methods find the nearest elements:-

1. lower() — Just Smaller Element of Given element.

2. higher() — Just Greater Element of Given element.

3. floor() — Equal element if available or smaller than given element.

4. ceiling() — Equal element if available or greater than the given element.

5. pollFirst() — Remove the first element from the set and return it.

6. pollLast() — Remove the last element from the set and return it.

7. descendingSet() — Return set in Reverse Order of Given TreeSet

8. descendingIterator() - Return an Iterator that provides iteration in reverse order.

		System.out.println(set.lower(10));
		System.out.println(set.higher(10));
		System.out.println(set.descendingSet());
	}
}

Queue:-
======
- Queue is an interface in the Java Collection Framework.
- It is used to store elements in FIFO order (First In First Out).
- The element that is inserted first will be removed first.
- Queue allows duplicate elements.
- Queue generally does not allow null values (most implementations).
- It is mainly used for processing elements in order.

ArrayDeque:-
=-=-=-=-=-=
- ArrayDeque is a class that implements the Deque interface.
- It allows insertion and deletion of elements from both ends.
- duplicates allowed.
- It does not allow null values.
- It maintains insertion order logically.
- Internally uses a Resizable Circular Array.
- It is faster than LinkedList when used as Stack or Queue.
- Implements:
	- Deque<E>
	- Queue<E>
	- Iterable<E>
	- Cloneable
	- Serializable

Internal Working of ArrayDeque
-----------------------------------
- ArrayDeque is internally implemented using an Object[] array.
- It works using a circular array mechanism.
- Two important indexes are maintained internally:
	- head → points to first element
	- tail → points to next empty position

How Elements Are Stored:-

- When an element is added at the rear, it is stored at:
		elements[tail] = ele;
	After storing:
		tail = (tail + 1) % capacity

- When an element is added at the front, first head moves:
		head = (head - 1 + capacity) % capacity
	Then element is stored:
		elements[head] = ele;

How Elements removed:-

- When element remove at the rear:
		tail = (tail - 1 + capacity) % capacity
		result = elements[tail]
		elements[tail] = null

- When element remove at the front:
		result = elements[head]
		elements[head] = null;
		head = (head + 1) % capacity

- Iteration logic:-
	index = head
	while(index != tail) {
    		print(elements[index])
    		index = (index + 1) % capacity
	}

PriorityQueue:-
=-=-=-=-=-=-=
- PriorityQueue is a class that implements the Queue interface.
- It allows duplicate elements.
- It does not allow null values.
- It does not maintain insertion order.
- Elements are stored based on priority order.
- By default, elements are arranged in natural ascending order.
- Internally uses a Binary Heap (Min Heap).
- The smallest element always remains at the front (root).
- It is mainly used when priority-based processing is required.
- Implements:
	- Queue<E>
	- Iterable<E>
	- Serializable
	- Cloneable

Internal Working of PriorityQueue
-----------------------------------
- It is internally implemented using a Binary Heap.
- By default, it uses a Min Heap structure.
- In Min Heap, parent element is always smaller than its children.
- PriorityQueue internally stores elements in an array.
- When an element is added, it is first inserted at the end of the array.
- After insertion, UpHeapify (Bubble Up) process is performed to maintain heap property.
- During UpHeapify:
	- The inserted element is compared with its parent.
	- If the element is smaller than parent, swapping happens.
	- This process continues until heap property is satisfied.
- When an element is removed using poll():
	- The root element (smallest) is removed.
	- The last element is moved to root position.
	- DownHeapify (Bubble Down) process is performed.
- During DownHeapify:
	- Root element is compared with its children.
	- It is swapped with the smaller child.
	- This continues until heap property is restored.
- The root element is always stored at index 0 of the array.

Index Relation in Heap:
-------------------------
- Parent Index = Floor((i - 1) / 2)
- Left Child Index = (2 * i) + 1
- Right Child Index = (2 * i) + 2



Min Heap Tree (Example)
=-=-=-=-=-=-=-=-=-=-=
Rule: Parent node is smaller than or equal to its children.

Example Numbers: 5, 10, 20, 30, 40, 50, 35

             5
         /      \
      10          20
    /    \      /       \
 30      40   50        35

Array Representation:

[5, 10, 20, 30, 40, 50, 35]


Max Heap Tree (Example)
=-=-=-=-=--=-=-=-=-=

Rule: Parent node is greater than or equal to its children.

Example Numbers: 50, 30, 40, 10, 5, 20, 35

              50
          /       \
        30        40
    /      \     /    \
 10         5   20     35

Array Representation:

[50, 30, 40, 10, 5, 20, 35]







Default Capacity and Growth:-
=-=-=-=-==-=-=-=-=-=-=-=-=-=
- Default initial capacity: 11
- Capacity increases automatically when elements exceed current capacity.
- Capacity grows dynamically as elements are added.
- After resizing new Array capacity is:-
	New Size = 1.5 × Old Size
- During resizing, a new larger array is created.
- Existing elements are copied into the new array.
- Heap structure is maintained after resizing.

Internal Working of HashMap
---------------------------
- HashMap stores data in the form of key-value pairs.
- When a key-value pair is added, the hashCode() method of the key is called to generate hashCode.
- The hashCode() method generates an integer value that helps determine the bucket location.
- Based on this hashCode, the key-value pair is placed into a specific bucket.
- If the bucket is empty, the key-value pair is stored directly.
- If the bucket already contains elements, collision occurs.
- In case of collision, equals() method is used to compare the new key with existing keys in that bucket.
- If equals() returns true, the old value is replaced with new value.
- If equals() returns false, the new entry is added into the same bucket.

Important Points:
- hashCode() is always checked first to find bucket location.
- equals() is checked only when multiple keys have same hashCode.
- If both hashCode is same and equals returns true → value replaced.
- If hashCode same but equals returns false → new entry added.

Collision Handling (Java 8+)
- Before Java 8 → Collision handled using Linked List.
After Java 8 →
- If number of elements in bucket > 8
- Linked List converted into Balanced Tree (Red-Black Tree).
- This improves performance.

Default Capacity and Load Factor
--------------------------------
- Default initial capacity: 16
- Load factor: 0.75 (When number of elements exceeds 75% of capacity, resizing happens.)

Example:

16 × 0.75 = 12 (After 12 entries → resizing occurs.)

Capacity Expansion:
--------------------
- Capacity is doubled after reaching load factor limit.

Example:
16 → 32 → 64 → 128

- During resizing, all existing entries are rehashed.
- Rehashing means:
	- Recalculate bucket location
	- Place entries into new buckets

- This helps maintain efficient performance.

Fail fast and Fail safe behaviour:-
=-=-=-=-=-=-=-=-=-=-=-=--==-=-=-=-=
Fail-Fast Behaviour:-
------------------
- A Fail-Fast iterator throws an exception if the Map is structurally modified while iterating.

- Structural modification means:
	- Adding elements
	- Removing elements

- Except when modification is done using iterator's own remove() method.

- When such modification happens, Java throws:
	- ConcurrentModificationException

Example:-
--------
public class Main {
	public static void main(String[] args) {
		Map<Integer, String> map = new HashMap<>();
		map.put(1, "Aditya");
		map.put(2, "Rachit");
		map.put(3, "Raghav");
		map.put(4, "Ram");
		map.put(5, "Shyam");
		map.put(6, "Sita");
		for(Integer x : map.keySet() ) {
			if(x ==4) {
				map.remove(x);
			}
		}
		System.out.println(map);
	}
}

Fail-Safe behaviour:
---------------------
When we iterate over a Map, and the Map is modified at the same time, no exception is thrown, and the program continues normally.

2. LinkedHashMap:-
=-=-=-=-=-=-=-=-=
LinkedHashMap is a class that implements the Map interface.
- Most of the property are same as HashMap.
- The basic difference is:-
	- It maintains insertion order.
	- Internally uses Hash Table + Linked List.
- Implements:
	- Map<K,V>
	- Cloneable
	- Serializable

Internal Working of LinkedHashMap:-
-----------------------------------
- LinkedHashMap works similar to HashMap, but it also maintains insertion order.
- LinkedHashMap internally uses:
	Hash Table + Doubly Linked List
- When a key-value pair is added:
- hashCode() method of key is called.
- Bucket location is calculated.
- Entry stored in HashMap bucket.
- Entry also connected using Linked List to maintain order.

Default Capacity and Load Factor
---------------------------------
Same as HashMap:
- Default initial capacity: 16
- Load factor: 0.75

Example:

16 × 0.75 = 12 (After 12 entries → resizing occurs.)

Capacity Expansion:
- Capacity is doubled after reaching load factor.

Example:

16 → 32 → 64 → 128 (All entries are rehashed.)

Rehashing means:
- Recalculate bucket index
- Move entries to new buckets

Creation of LinkedHashMap:-
=-=-=-=-=-=-=-=-=-=-=-=-=

With type safety:

	Map<Integer,String> map = new LinkedHashMap<>();

With initial capacity:

	Map<Integer,String> map = new LinkedHashMap<>(20);

With capacity and load factor:

	Map<Integer,String> map = new LinkedHashMap<>(20,0.75f);

Methods:-
=-=-=-=
(Same methods as HashMap — because LinkedHashMap also implements Map interface.)

TreeMap:-
=-=-=-=-=
TreeMap is a class that implements the Map interface.
- It stores data in key-value pair format.
- It does not allow duplicate keys.
- It allows duplicate values.
- It maintains the sorted order of keys.
- It does not allow a null key.
- It allows multiple null values.
- It is not synchronized.
- Internally uses Red-Black Tree.
- It sorts keys using:
	Natural Sorting (Comparable)
	OR
	Custom Sorting (Comparator)
- Implements:
	NavigableMap<K,V>
	SortedMap<K,V>
	Map<K,V>
	Cloneable
	Serializable

Internal Working of TreeMap
----------------------------
- TreeMap stores data using Red-Black Tree.
- Red-Black Tree is a type of:
	- Self-Balancing Binary Search Tree
	- Keys are stored in sorted order.

- When a key-value pair is added:
- TreeMap compares keys.
- Finds the correct position.
- Inserts entry into tree.
- Tree re-balances automatically.
- TreeMap does not use hashing.
- TreeMap uses:
	compareTo() (Inside Comparable interface for Natural Sorting)
  	OR  
	compare() (Inside Comparator interface for Custom Sorting)
	(methods to compare keys)

- Elements are arranged based on:
	- Sorted order of keys

Default Capacity and Load Factor
--------------------------------
- TreeMap does not use capacity or load factor.

Because:
- It does not use hashing
- It uses tree structure

So:
- No resizing like HashMap

Creation of TreeMap:-
=-=-=-=-=-=-=-=-=-=-=
Deafault Constructor:-
	Map<Integer,String> map = new TreeMap<>();

With custom Comparator:
	Map<Integer,String> map = new TreeMap<>((a,b) -> b - a); //Uisng Lambda Expression

With existing Map:
	Map<Integer,String> map2 = new TreeMap<>(map1);

Hashtable:-
=-=-=-=-=

Hashtable is a class that implements the Map interface.
- It stores data in key-value pair format.
- It does not allow duplicate keys.
- It allows duplicate values.
- It does not maintain insertion order.
- It does not allow null key.
- It does not allow null value.
- It is synchronized (Thread-safe).
- It internally uses Hash Table.
- It is considered legacy class (old class from early Java versions).
- Implements:
	- Map<K,V>
	- Cloneable
	- Serializable

Internal Working of Hashtable:-
------------------------------
- Hashtable works similar to HashMap, but it is synchronized.


Default Capacity and Load Factor
-------------------------------
- Default initial capacity: 11
- Load factor: 0.75

Example:

11 × 0.75 ≈ 8 (After about 8 elements → resizing occurs.)

Capacity Expansion:
---------------------
- Capacity increases using:
	- (Old Capacity × 2) + 1

Example:
11 → 23 → 47 → 95
All existing entries are rehashed.

Creation of Hashtable:-
=-=-=-=-=-=-=-=-=-=-=-=

Default Constructor:-

	Map<Integer,String> map = new Hashtable<>();
